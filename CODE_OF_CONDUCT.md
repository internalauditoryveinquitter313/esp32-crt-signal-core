# Code of Conduct

## Our Pledge

We want ESP32 CRT Signal Core to be a useful, welcoming open source project for people working on embedded video, retro hardware, signal generation, tooling, documentation, and tests.

Everyone participating in this project is expected to act with respect, patience, and technical honesty.

## Expected Behavior

- Be respectful when discussing ideas, trade-offs, bugs, and implementation details.
- Keep feedback focused on the work, evidence, and project goals.
- Assume good intent, but be direct when something is unsafe, incorrect, or harmful to the project.
- Make space for contributors with different levels of experience in ESP-IDF, embedded C, analog video, and open source workflow.
- Credit prior work, references, tests, measurements, and hardware findings clearly.

## Unacceptable Behavior

- Harassment, insults, personal attacks, or discriminatory language.
- Publishing private contact information or private communications without consent.
- Repeatedly derailing technical discussions after maintainers have asked to refocus.
- Misrepresenting test results, hardware behavior, measurements, licenses, or security impact.
- Any conduct that would make reasonable contributors feel unwelcome or unsafe.

## Project-Specific Expectations

This project touches hardware timing, DMA, ISR behavior, and analog output. Technical disagreements are expected. Keep them grounded:

- Share reproduction steps, measurements, logs, captures, or code references when possible.
- Mark unverified hardware claims as unverified.
- Do not pressure maintainers to merge changes that weaken signal correctness, safety, or license clarity.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or block participants who violate this Code of Conduct. Enforcement should be proportionate to the impact and pattern of behavior.

To report conduct concerns, contact the maintainers privately through the project owner's GitHub profile or email `gabrielmaialva33@gmail.com`.

This Code of Conduct is inspired by common open source community standards, including the Contributor Covenant.
